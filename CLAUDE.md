## Project

Personal website for frankbuckley.com — static site with blog, biography, and CV. Built with Astro 6, deployed to GitHub Pages via `.github/workflows/deploy.yml` (targets Node 25).

## Commands

- `npm run dev` — local dev server
- `npm run build` — production build to `dist/`
- `npm run preview` — preview the production build

No tests, no linter. Type checking is via Astro's `strict` TypeScript preset.

## Architecture

- **Astro 6** single-file `.astro` components with file-based routing in `src/pages/` (home, about, cv, blog index, `blog/[slug]` dynamic route, `rss.xml.ts`).
- **Layouts**: `src/layouts/Layout.astro` is the base HTML shell (nav/main/footer, optional `title` prop). `src/layouts/BlogPost.astro` wraps `Layout` for posts (title/date/tags/content).
- **Content collections**: blog posts live as Markdown in `src/content/blog/`, schema in `src/content.config.ts`. Filename (minus `.md`) is the slug → `/blog/<slug>/`.
- **Styling**: Bootstrap 5 (CSS only, no JS bundle) + Bootstrap Icons icon font. Global styles and theme in `src/styles/default.css`. Blog Markdown renders inside a `.prose` wrapper that styles headings, paragraphs, lists, links, code blocks.
- **RSS** generated by `@astrojs/rss` at `/rss.xml`.
- **Static assets**: `public/` served as-is.

## Conventions

- `trailingSlash: "always"` in `astro.config.mjs` — **all internal links must end with `/`**.
- Use Bootstrap utility classes and `<i class="bi bi-*">` for icons. Reference theme colours via CSS vars: `--color-text` (`#1a1a1a`), `--color-text-muted` (`#555`), `--color-accent` (`#2563eb`), `--color-border` (`#e5e5e5`).
- Pages import and wrap content in `Layout`.
